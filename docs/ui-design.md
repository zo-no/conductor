# UI 设计

## 产品定位

Conductor 的 UI 是任务管理界面，面向人类用户。AI 通过 CLI 操作任务，人类通过 UI 查看、审批、干预。

---

## 整体布局

### PC 端

三栏布局：项目列表 + 时间线 + 任务编辑侧边栏

```
┌──────────┬─────────────────────┬──────────────┐
│ 项目列表  │ 时间线               │ 任务编辑      │
│          │                     │（点击任务展开）│
│ 日常事务● │ 今天                 │              │
│ 工作项目  │ 09:00 ↻ 每日晨报    │              │
│ 读书计划  │ 10:30 ○ 确认预算    │              │
│          │                     │              │
│ + 新建    │ 周期任务 ▶           │              │
└──────────┴─────────────────────┴──────────────┘
```

- 任务编辑从右侧滑出，不遮挡时间线
- 未选中任务时右侧区域为空

### 移动端

顶部 tab 切换 human / ai，各自内部为时间线列表

```
┌─────────────────────────────────────┐
│ [日常事务] [工作项目] [读书] →        │  ← 横向滚动项目 tab
├─────────────────────────────────────┤
│ [人类] [AI]                          │  ← assignee tab，默认 人类
├─────────────────────────────────────┤
│                                     │
│  时间线任务列表                       │
│                                     │
└─────────────────────────────────────┘
```

- 点击任务 → 全屏编辑页
- 默认显示人类 tab

## 时间线

PC 端 human + AI 任务统一时间线，移动端按 tab 过滤 assignee，结构相同。

### 分区规则

```
今天
─────────────────────────────────
09:00  ↻ 每日晨报（AI · running）
10:30  ○ 确认预算（人类 · 待处理）  ← 红色

明天 / 未来日期
─────────────────────────────────
09:00  ↻ 每日晨报（AI · pending）

周期任务 ▶（默认折叠）
─────────────────────────────────
  ↻ 每周回顾   下次：周一 09:00
  ↻ 每日晨报   下次：明天 09:00

无时间
─────────────────────────────────
  ⏸ 竞品调研（AI · blocked）
      └─ ○ 确认预算  ← 红色，可直接勾选
  ○ 审核报告（人类 · 待处理）

已完成（折叠）
─────────────────────────────────
```

### 排序规则

- 有时间的任务：按 `scheduledAt` / `nextRunAt` 升序
- `kind=recurring`：进"周期任务"折叠区，显示 `nextRunAt`
- `kind=once` 无 `scheduledAt`：进"无时间"区，置于底部
- `enabled=false`：灰色 + "已暂停"标签，仍在列表中

### blocked 任务展示

blocked AI 任务在"无时间"区，内联显示关联的 human 任务：

```
⏸ 竞品调研（AI · blocked）
    └─ ○ 确认预算  ← 可直接勾选完成
```

- 数据来源：`blockedByTaskId` → human 任务，`sourceTaskId` 反向关联
- human 任务勾选完成后，AI 任务自动恢复 pending，动画过渡

## 状态视觉映射

### AI 任务

| 状态 | 视觉 |
|---|---|
| pending | 灰色空心圆 |
| running | 绿色实心点（波动动画） |
| blocked | 橙色 ⏸ |
| failed | 红色 ✗ |
| done | 灰色 ✓ |
| cancelled | 灰色删除线 |

### 人类任务

| 状态 | 视觉 |
|---|---|
| pending | 红色空心圆（可勾选） |
| done | 灰色 ✓ |
| cancelled | 灰色删除线 |

## 任务编辑面板

PC 端从右侧滑出，移动端全屏页。字段按 assignee + kind 组合显示。

### 所有任务共有字段

- 标题（必填）
- 描述（可选）
- 归属项目
- 启用/禁用 toggle

### AI 任务字段

- 执行器类型（ai_prompt / script / http）
- **ai_prompt**：prompt 文本（支持占位符：`{date}`、`{datetime}`、`{projectName}`、`{lastOutput}` 等）
- **script**：Shell 命令 + 工作目录
- **http**：URL + Method + Headers + Body
- 执行服务（tool / model 选择）
- 是否注入上次执行结果（includeLastOutput toggle）
- 执行完创建 review 任务（reviewOnComplete toggle）

### 调度字段（kind=scheduled/recurring 时显示）

- `scheduled`：定时时间（datetime-local）
- `recurring`：cron 表达式 + 时区（提供常用模板：每天/每周/每小时）

### 人类任务字段

- 完成说明（waitingInstructions，只读，AI 填写）
- 来源任务（只读，sourceTaskId 关联）

## 项目管理

### 项目列表（侧边栏）

```
▼ 我的项目
  日常事务  ●
  工作项目
  读书计划
  ──────────
  + 新建项目
```

- `●` 表示有人类待处理任务
- 归档项目折叠在底部，默认不显示

### 项目设置（点击项目名称旁的 ⚙）

- 项目名称
- 目标描述（goal，注入 AI 上下文）
- 工作目录（workDir，AI 执行任务的默认目录）
- 项目级系统 prompt（systemPrompt）
- 归档 / 删除
