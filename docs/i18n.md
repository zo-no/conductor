# 国际化（i18n）

Conductor 前端使用轻量级自研 i18n 方案，无第三方依赖，支持中文（`zh`）和英文（`en`）。

---

## 架构

```
packages/web/src/lib/i18n.ts   # 翻译表 + hook + 工具函数
```

所有翻译字符串集中在一个文件里，分 `zh` 和 `en` 两个对象。支持：

- 普通字符串：`t('save')` → `'保存'` / `'Save'`
- 带参数的函数：`t('deleteN', 3)` → `'删除 3 项'` / `'Delete 3'`
- 模板字符串：`t('confirmDeleteTask', task.title)`

---

## 使用方法

### 在 React 组件里

```tsx
import { useT } from '../lib/i18n'

function MyComponent() {
  const t = useT()

  return (
    <button>{t('save')}</button>
  )
}
```

### 在组件外（工具函数、常量等）

```ts
import { t } from '../lib/i18n'

const label = t('newTask')
```

### 切换语言

```tsx
import { useLocale } from '../lib/i18n'

function LangToggle() {
  const [locale, setLocale] = useLocale()

  return (
    <button onClick={() => setLocale(locale === 'zh' ? 'en' : 'zh')}>
      {locale === 'zh' ? 'English' : '中文'}
    </button>
  )
}
```

当前语言存储在 `localStorage` 的 `conductor_locale` key 下，页面刷新后保持。

---

## 添加新翻译

在 `packages/web/src/lib/i18n.ts` 的 `translations` 对象里，同时在 `zh` 和 `en` 两个 section 添加同名 key：

```ts
const translations = {
  zh: {
    // ...
    myNewKey: '新字符串',
    myNewKeyWithArg: (name: string) => `你好，${name}`,
  },
  en: {
    // ...
    myNewKey: 'New string',
    myNewKeyWithArg: (name: string) => `Hello, ${name}`,
  },
}
```

TypeScript 会自动推断类型，如果 `zh` 和 `en` 的 key 不一致会报类型错误。

---

## 语言检测

首次访问时按以下顺序确定语言：

1. `localStorage.conductor_locale`（用户手动切换后保存）
2. `navigator.language`（浏览器语言，`zh` 开头 → 中文，否则英文）

---

## 当前支持的语言

| 语言 | 代码 | 状态 |
|------|------|------|
| 中文（简体）| `zh` | 完整 |
| English | `en` | 完整 |

---

## 切换入口

PC 端：侧边栏底部有语言切换按钮（展开时显示文字，折叠时显示 `EN` / `中`）。
