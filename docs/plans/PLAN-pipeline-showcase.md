# Showcase: 流水线人机协作场景

> 状态: DRAFT（讨论中）
> 创建时间: 2026-04-16
> 目的: 用具体 case 跑通人机协作闭环，找出卡点

---

## Showcase 1：AI 写代码 → 用户 review → AI commit

**痛点**：AI 写完代码不知道该不该提交，用户不知道 AI 在等自己。

**流程**：

```
[AI 任务] 实现登录功能
    ↓ 完成后
[Human Todo] "请 review 这个 diff，approve 后我会 commit"
    ↓ 用户点完成
[AI 任务] git commit && push
```

**需要验证**：
- [ ] `--review-on-complete` 触发下一个 AI 任务时，AI 能拿到 human todo 的内容吗？
- [ ] 用户有没有地方填"修改意见"？AI 能读到吗？

---

## Showcase 2：AI 遇到分叉 → 用户选择 → AI 继续

**痛点**：AI 分析完发现有两种方案，不知道选哪个，会话就卡住了。

**流程**：

```
[AI 任务] 分析重构方案
    ↓ 发现两种路径
[Human Todo] "请选择方案：A（激进重构）或 B（渐进迁移）"
    ↓ 用户填写 "A" 或 "B"
[AI 任务] 按用户选择执行对应方案
```

**需要验证**：
- [ ] 用户填写的文本，下一个 AI 任务怎么读取？
- [ ] 是否需要 todo 支持结构化选项（而不是自由文本）？

---

## Showcase 3：AI 需要凭证 → 用户提供 → AI 继续

**痛点**：AI 执行到一半发现需要 API Key，只能停下来等，用户不知道。

**流程**：

```
[AI 任务] 配置第三方服务集成
    ↓ 需要 API Key
[Human Todo] "请提供 Stripe API Key（生产环境）"
    ↓ 用户填写 key
[AI 任务] 写入配置文件，继续集成
```

**需要验证**：
- [ ] 用户填写的敏感信息如何安全传递给 AI？
- [ ] AI 任务拿到内容后怎么用（注入 context？写文件？）

---

## Showcase 4：长任务链中的检查点

**痛点**：自动化流水线跑到"部署"这步，用户希望手动确认一次再放行。

**流程**：

```
[AI 任务] 跑测试
    ↓ 自动
[AI 任务] 构建产物
    ↓ 自动
[Human 检查点] "测试通过，构建完成，确认部署到生产？"
    ↓ 用户确认
[AI 任务] 部署到生产
```

**需要验证**：
- [ ] 任务链怎么定义？（现在 conductor 有任务依赖关系吗？）
- [ ] 检查点是一种特殊的 human todo，还是需要新的任务类型？

---

## 当前最小可跑通的 Case

**推荐从 Showcase 1 开始**，原因：
- 不需要传递用户输入内容，只需要"approve"信号
- `--review-on-complete` 已有，最接近可用
- 流程最清晰，容易验证

**验证步骤**：
1. 手动创建一个 human todo，带 `--review-on-complete`，指向一个 AI 任务
2. 在 web UI 或 CLI 完成这个 todo
3. 观察 AI 任务是否自动触发
4. AI 任务里能不能读到 todo 的 instructions 内容
