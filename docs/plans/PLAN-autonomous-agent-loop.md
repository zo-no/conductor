# 范式：自治 Agent 循环（Autonomous Agent Loop）

> 状态: DRAFT
> 创建时间: 2026-04-16
> 来源: 理财 case 实践总结

---

## 核心思想

**给 AI 一个长期目标，AI 自主规划、派发任务、周期迭代。**

Todo/任务不是工作本身，而是 AI 和人之间的**消息通道**。

```
[长期目标] → [月度规划 AI 任务（recurring）]
                    ↓ 每个周期执行
              读取项目当前状态
                    ↓
              决策：本周期要做什么
                    ↓
         ┌──────────────────────┐
         ↓                      ↓
   [派发 human task]      [派发 ai task]
   "请更新收支数据"        "分析上月支出趋势"
         ↓                      ↓
   用户完成后继续          AI 执行后输出结果
         └──────────────────────┘
                    ↓
              下一个周期继续
```

---

## 结构模板

每个使用此范式的项目包含：

### 1. Project（项目容器）
- `name`: 项目名称
- `goal`: **详细的长期目标**（这是 AI 每次规划的基准）
- `workDir`: AI 执行时的工作目录

### 2. 规划任务（心跳任务）
- `assignee`: ai
- `kind`: recurring
- `recurringConfig.cron`: 根据项目节奏设置（每日/每周/每月）
- `executor.kind`: ai_prompt
- `executor.prompt`: 见下方 Prompt 模板

### 3. 规划 Prompt 模板

```
你是「{项目名}」项目的{周期}规划 Agent。

## 项目目标
{从 project.goal 复制，详细描述}

## 你的职责
每{周期}执行一次，完成以下工作：

### 第一步：读取当前状态
运行：conductor task list --project {projectId} --json
分析：哪些任务完成了？哪些卡住了？距离目标进展如何？

### 第二步：规划本{周期}行动
根据项目目标和当前状态，决定本{周期}需要做什么。

### 第三步：派发任务
- 需要用户做的事 →
  conductor task create --project {projectId} --assignee human --kind once \
    --title "..." --waiting-instructions "..."
- 需要 AI 执行的分析/操作 →
  conductor task create --project {projectId} --assignee ai --kind once \
    --executor ai_prompt --prompt "..."

### 固定检查项（每次都要做）
{列出每个周期必须检查的事项}

### 输出
执行完所有 conductor 命令后，输出本{周期}规划摘要（不超过200字）。
```

## 关键设计原则

### 1. 规划任务只做规划，不做执行
规划任务的职责是**读状态 → 做决策 → 派任务**，具体工作由派发出去的子任务完成。

### 2. project.goal 是 AI 的长期记忆
每次规划时 AI 都会读 goal，所以 goal 要写得足够具体：
- 包含量化指标（"每月净存1万"而不是"多存钱"）
- 包含时间节点（"5年后买房"）
- 包含约束条件（"风险偏好平衡"）

### 3. human task 是通知机制
AI 创建 human task = 给用户发消息。用户完成 = 确认/提供信息。

### 4. 可以手动触发跑第一次
创建好规划任务后，立即手动触发一次（`conductor task run <id>` 或 API），
不用等到下一个 cron 时间点。

## 已验证的 Case

### Case 1：个人理财（2026-04-16）

**目标**: 月收入2万，支出8千，5年后买房（60万首付）

**结构**:
- Project: `proj_da45e498fecb`（理财规划）
- 规划任务: `task_6912bdd79e02`（月度理财规划，每月1号9点）

**规划任务职责**:
1. 读取项目任务状态
2. 提醒用户更新上月收支数据（human task）
3. 对比储蓄目标进度
4. 安排必要的分析任务

**迭代方向**:
- 第1个月：建立基础数据收集习惯
- 第2-3个月：引入支出分类分析
- 第4-6个月：加入投资追踪
- 第7-12个月：年度复盘，调整策略

## 适用场景

| 场景 | 周期 | 说明 |
|------|------|------|
| 个人理财 | 月度 | 收支追踪、储蓄目标 |
| 健康管理 | 周度 | 运动、饮食、睡眠 |
| 学习计划 | 周度 | 进度追踪、调整计划 |
| 项目管理 | 周度/双周 | Sprint 规划 |
| 代码库维护 | 月度 | 技术债、依赖更新 |

## 开放问题

| 问题 | 优先级 |
|------|--------|
| AI 创建的任务如何引用上一个任务的输出？ | 高 |
| 规划任务能读取已完成 human task 的用户回复吗？ | 高 |
| 如何防止规划任务重复创建相同的子任务？ | 中 |
| project.goal 能否支持结构化字段（而不是纯文本）？ | 低 |
